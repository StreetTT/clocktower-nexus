# Packages

The `packages/` workspace contains shared modules used across apps in the modular monolith.

- `domain`: canonical game state, commands, events, reducers, and projections
- `protocol`: shared HTTP and WebSocket contracts
- `script-import`: script ingestion helpers
- `script-normalization`: normalized internal script model helpers
- `script-compatibility`: compatibility helpers for external or community script formats

These workspaces are intentionally placeholders in Task 1.1.2. Internal structure and code land in later tasks.
